Which one do I need?

  • Shamir — simple backup/recovery of a password, key, or seed phrase. Any k of n trusted holders can recover it.
  • VSS — when the party creating the shares might be dishonest, or shares might be corrupted, and you need to detect it before combining.

How it works

The secret becomes the constant term of a random polynomial of degree k−1; shares are points on the curve. Any k points determine the polynomial (and the secret) via Lagrange interpolation; k−1 points reveal nothing. Shamir works byte-by-byte in the GF(256) field; VSS works in a 2048-bit prime field and adds g^coefficient commitments so each share is verifiable.

Both tools produce the same wire format as the command-line apps in this repository, so a share created on the web can be combined in the CLI and vice versa.

Educational project. Source and command-line versions are in the repository. For production use, prefer an audited library (e.g. SLIP-0039 implementations).
